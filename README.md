# 📊 LoanLens（ローンレンズ）

## サービス概要

複数の住宅ローン条件（金利、期間、繰り上げ返済）のシミュレーション結果を、1画面でシームレスに同時比較・可視化できる、モダンな**「階段グラフ特化型ローンシミュレーター」**です。

既存の銀行サイトのようなページ遷移や複雑な元利内訳の表を排除し、「スライダーを動かした瞬間に、将来の返済額の変動リスクが直感的にわかる」という圧倒的なユーザー体験（UX）を提供します。

## このサービスへの思い・作りたい理由

家を買うという人生最大の決断において、多くの人が「変動金利のリスク」や「当初固定金利が終わった後の返済額の跳ね上がり」に不安を抱えています。
しかし、既存のシミュレーターは「35年分の細かい数字の表」を読ませるものが多く、直感的にリスクを理解するのが困難です。

また開発者として、実務で主流となっている**「フロントエンド（Next.js）とバックエンド（Rails API）の完全分離（モノリポ構成）」**に挑戦し、複雑なドメイン知識（金利計算・繰り上げ返済ロジック）を**TDD（テスト駆動開発）**で堅牢に実装する経験を積むためにこのサービスを企画しました。

## ユーザー層について

**メインターゲット**：

- 住宅ローンの借入を検討中で、変動金利・固定金利のどちらにすべきか悩んでいる方
- 将来的に繰り上げ返済（期間短縮 / 返済額軽減）を計画しており、その効果をビジュアルで確認したい方
- 銀行の小難しいシミュレーターではなく、手元でサクサク条件を動かして比較したい方

## サービスの利用イメージ

1. トップ画面（1画面ダッシュボード）にアクセスする。
2. 左側のコントロールパネルで、「借入金額」や「借入期間」をスライダーで直感的に調整する。
3. 「当初5年固定金利（当初0.7%、終了後1.5%）」のような複雑な金利プランをポチポチと入力する。
4. 「5年目に100万円を繰り上げ返済（期間短縮型）」のトグルスイッチをONにする。
5. 右側のグラフの右端が「キュッと縮む（完済が早まる）」動きを視覚的に確認し、最適な返済計画を見つける。

**得られる価値**：

- **金利上昇リスクの可視化**：固定期間が終わった瞬間に返済額がカクッと上がるポイントがひと目でわかります。
- **繰り上げ返済効果の体感**：お金を返した瞬間に、グラフが縮む（期間短縮）、あるいはグラフが下がる（返済額軽減）という直感的なフィードバックが得られます。

## サービスの差別化ポイント・推しポイント

### **類似サービスとの差別化**

- **1画面・3シナリオの同時比較**：ページ遷移をせず、同じ画面内で「変動」「全固定」「当初固定」の3つのカードの数字とグラフの線がリアルタイムに連動します。
- **「返済額」に特化したステップチャート**：内訳（元本と利息の細かい内訳）の表示を削ぎ落としたことで、ユーザーが一番知りたい「毎月いくら払うのか」の推移に100%集中できるUIにしています。

### **推しポイント**

1. **実務レベルのモノリポ（Monorepo）構成**：1つのリポジトリ内で `frontend` と `backend` を綺麗に分離し、VercelとRenderにそれぞれ最適化して自動デプロイします。
2. **堅牢なModel TDD**：浮動小数点のバグが発生しやすい金利計算ロジックを、Railsのモデル内にカプセル化し、RSpecを用いたTDD（テスト駆動開発）で100%の計算精度を保証しています。
3. **洗練されたApple風 minimal UI**：Tailwind CSSを用い、背景が薄く透ける「ガラスモルフィズム（Glassmorphism）」を取り入れた、金融ツールとは思えない洗練されたデザイン。

## 機能一覧

- [ ] 3シナリオ同時比較ダッシュボード（1画面完結）
- [ ] 借入金額・借入期間のリアルタイムスライダー変更機能
- [ ] 当初期間固定金利シミュレーション（期間、当初金利、終了後想定金利の入力）
- [ ] 繰り上げ返済シミュレーション（実行年、金額、期間短縮型/返済額軽減型の選択）
- [ ] Rechartsを用いた階段状（ステップ）の返済額推移グラフ描画
- [ ] 浮動小数点誤差を防ぐ整数型（万分率）による金利計算ロジック（バックエンド）

## 使用する技術スタック

| カテゴリ         | 技術                         | 採用理由                                                                                                    |
| ---------------- | ---------------------------- | ----------------------------------------------------------------------------------------------------------- |
| バックエンド     | Ruby on Rails 8.x / Ruby 3.x | APIモードとして採用。強力なActive Recordを用いて軽量かつ高速な計算サーバーを構築。                          |
| フロントエンド   | Next.js (TypeScript)         | App Routerを採用。Rechartsと組み合わせ、ぬるぬる動く高度な状態管理（State）とインタラクティブなUXを実現。   |
| フロントエンド   | Tailwind CSS                 | ガラスモルフィズム調のミニマルで洗練されたモダンUIを高速にコーディングするため。                            |
| データベース     | Neon（PostgreSQL）           | サーバーレスPostgreSQL。Rails 8との相性が最高であり、実務で注目されている「DBブランチ機能」を活用するため。 |
| インフラ (Front) | Vercel                       | Next.jsとネイティブに連携し、GitHub連携による爆速な静的配信・ビルドを行うため。                             |
| インフラ (Back)  | Render                       | 無料枠でRails APIコンテナ（Docker）をシンプルにホスティング・運用するため。                                 |
| グラフ           | Recharts                     | React専用の強力なチャートライブラリ。階段状グラフ（`type="stepAfter"`）を美しく表現可能。                   |
| コンテナ環境     | Docker / Docker Compose      | `ruby:3.x-slim` を使い、APIモードに特化したNode.js無しの超軽量な開発環境を統一するため。                    |
| テスト           | RSpec                        | ローン計算の正確性を担保するため、モデルに対する厳格なTDD（テスト駆動開発）を実施。                         |

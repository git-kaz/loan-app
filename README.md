## サービス概要
複数の住宅ローン条件（金利、期間、繰り上げ返済）のシミュレーション結果を、1画面でシームレスに同時比較・可視化できる、モダンな**「階段グラフ特化型ローンシミュレーター」**です。
スライダーを動かした瞬間に、将来の返済額の変動リスクが直感的にわかります。

---

## 🏠 開発背景と解決したい課題

### 背景（原体験）
私自身が実際にマイホームを購入する際、人生最大の決断である「住宅ローンの選定」にあたって、変動金利・固定金利の選択や、将来の繰り上げ返済のタイミングなど、無数のパターンをシミュレーションするために膨大な時間と労力を費やしました。
その際、既存の有名なシミュレーションツールをいくつか利用しましたが、それぞれに一長一短があり、大きなストレスを感じたことが本アプリの開発のきっかけです。

### 既存サービスの分析と課題
1. **[SUUMO 住宅ローンシミュレーター](https://suumo.jp/sp/apps/loansimulator/index.html)**
   * **良かった点**: UIが見やすく、スライダー操作でリアルタイムに毎月の返済金額の概算がわかるUXが非常に使いやすかった。
   * **不満だった点**: 1つのプランしか表示できず、**「複数の条件を比較できない」**ため、画面を行き来する必要があった。
2. **[住宅ローンシミュレーション - mamoris](https://loan.mamoris.jp)**
   * **良かった点**: 固定金利期間の設定など、非常に細かい実務的なローン設定ができる。
   * **不満だった点**: やラー複数の比較ができない。さらに**「UIが使いづらく、リアルタイムに結果が表示されない」**ため、数値を変更するたびに思考が遮断されてしまった。

### 解決アプローチ（本アプリのコンセプト）
既存ツールの良いとこ取りをし、**「SUUMOのような優れたUIとリアルタイム性」**と、**「mamorisのような実務的な細かい設定」**を両立した上で、さらに**「3つのプランを1画面で同時に比較・可視化できる」**という、これまでにないローンシミュレーター「**LoanSchumy**」を開発しました。

---

## 👤 ユーザー層について

**メインターゲット**：
* 住宅ローンの借入を検討中で、変動金利・固定金利のどちらにすべきか悩んでいる方
* 将来的に繰り上げ返済（期間短縮 / 返済額軽減）を計画しており、その効果をビジュアルで比較したい方
* 銀行の小難しいシミュレーターではなく、手元でサクサク条件を動かして「複数プランを並べて」比較したい方

---

## 🔥 サービスの推しポイント（他アプリとの差別化）

1. **1画面・3シナリオのリアルタイム同時比較**
   * ページ遷移をせず、同じ画面内で「変動」「全固定」「当初固定」などの3つのプランの数字とグラフの線がリアルタイム（ミリ秒単位）に連動します。
2. **「返済額」に特化したステップチャート（階段状グラフ）**
   * Rechartsを用い、返済額の内訳（元本と利息の細かい数値）をあえて削ぎ落とすことで、ユーザーが一番知りたい「毎月実際にいくら払うのか（金利上昇時にいつ跳ね上がるのか）」の推移に集中できるUIを構築。
3. **堅牢な計算ロジック（TDDの実践）**
   * 浮動小数点の誤差が発生しやすい複雑な金利計算や繰り上げ返済ロジックを、Railsのモデル内にカプセル化（Fat Model）。RSpecを用いたTDD（テスト駆動開発）により、100%の計算精度を保証しています。

---

## ✨ ユーザーの利用イメージ

1. **基本条件の入力**: 左側のコントロールパネルで、「借入金額」や「借入期間」をスライダーで直感的に調整します。
2. **金利プランの設定**: 「当初3年固定金利（当初0.9%、終了後1.5%）」のような複雑な実務的プランを設定します。
3. **繰り上げ返済の試算**: 「5年目に100万円を繰り上げ返済（期間短縮型）」のトグルスイッチをONにします。
4. **リアルタイムな比較**: 右側の3つのカードで毎月の返済額の差を確認しつつ、下部のグラフが「キュッと縮む（完済が早まる）」動きを視覚的に確認し、最適な返済計画を見つけます。

## 📝 機能一覧と使用技術

### 主要機能
- [x] 3シナリオ同時比較ダッシュボード（1画面完結）
- [x] 借入金額・借入期間のリアルタイムスライダー変更機能
- [x] 当初期間固定金利シミュレーション（期間、当初金利、終了後想定金利の入力）
- [x] 繰り上げ返済シミュレーション（実行年、金額、期間短縮型/返済額軽減型の選択）
- [x] Rechartsを用いた階段状（ステップ）の返済額推移グラフ描画
- [x] 浮動小数点誤差を防ぐ整数型（万分率）による金利計算ロジック（バックエンド）
- [x] GitHub ActionsによるCI（RuboCop、RSpec、ESLintの実行）

### 技術スタック

| カテゴリ | 技術 | 役割・採用理由 |
| :--- | :--- | :--- |
| バックエンド | **Ruby on Rails 8.x (Ruby 3.x)** | APIモード。Active Recordを用いた高速なローン計算サーバー。 |
| フロントエンド | **Next.js (TypeScript / App Router)** | ぬるぬる動く高度な状態管理（State）とインタラクティブなUX。 |
| スタイリング | **Tailwind CSS** | ガラスモルフィズム調のミニマルで洗練されたモダンUIの構築。 |
| データベース | **Neon（PostgreSQL）** | サーバーレスPostgreSQL。開発中のDBブランチ機能を活用。 |
| グラフ描画 | **Recharts** | 階段状グラフ（`type="stepAfter"`）の滑らかな描画。 |
| コンテナ環境 | **Docker / Docker Compose** | 開発環境の完全なコンテナ化と軽量な実行環境の統一。 |
| テスト | **RSpec** | ローン計算ロジックに対する厳格なTDD（テスト駆動開発）の実施。 |
